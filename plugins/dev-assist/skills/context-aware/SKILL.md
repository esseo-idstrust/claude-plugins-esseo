## 자동 컨텍스트 수집

사용자가 커밋, PR, 코드 리뷰를 요청하면 다음 정보를 자동으로 수집합니다:

### 수집 항목

1. **현재 브랜치**: `git branch --show-current`
2. **스테이징 상태**: `git status --short`
3. **스테이징된 변경사항**: `git diff --staged --stat`
4. **최근 커밋 3개**: `git log --oneline -3`
5. **원격 브랜치와의 차이**: `git log origin/main..HEAD --oneline` (해당하는 경우)

### 사용 방법

이 스킬이 활성화되면 위 정보를 바탕으로:
- 커밋 요청 시: 스테이징된 변경사항을 분석하여 메시지 제안
- PR 요청 시: 브랜치 간 차이를 분석하여 PR 본문 작성
- 리뷰 요청 시: 변경 파일 목록을 파악하여 집중 리뷰 수행
